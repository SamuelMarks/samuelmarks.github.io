    A Deterministic, Specification-Driven Transpiler for Deep Learning Frameworks.

    ml-switcheroo is a rigorous AST-based transpiler designed to convert Deep Learning code between frameworks (e.g., * PyTorch* \(\leftrightarrow\) JAX, Keras \(\to\) TensorFlow) without hallucination.

    It uses a Hub-and-Spoke architecture to solve the \(O(N^2)\) translation problem. Instead of writing translators for every pair of frameworks, ml-switcheroo maps all frameworks to a central Abstract Standard (Hub). This allows for â€œZero-Editâ€ support for new frameworks via isolated JSON snapshots (Spokes).

    • ğŸ”Œ Hub-and-Spoke Architecture: Decouples the semantic definition of an operation (e.g., Conv2d) from its implementation (e.g., torch.nn.Conv2d).

    • ğŸ‘» Ghost Mode: Can analyze and transpile code for frameworks not installed on the local machine using cached API snapshots.

    • ğŸ›¡ï¸ Safety Logic: Automatically detects side-effects (IO, globals) that break functional compilation (JIT) via the Purity Scanner.

    • ğŸ§¬ Structural Rewriting: Handles complex transformations for class hierarchies (e.g., nn.Module \(\leftrightarrow\) flax.nnx.Module), random number threading, and state management.

    ğŸ› ï¸ CLI UsageÂ¶

    The ml_switcheroo tool provides a suite of commands for conversion, auditing, and knowledge base maintenance.

    1. Transpilation (convert)Â¶

    Convert a file or directory from one framework to another.

    # Convert a PyTorch model to JAX (Flax NNX)
ml_switcheroo convert ./models/resnet.py \
    --source torch \
    --target jax \
    --out ./resnet_jax.py

# Convert an entire directory, enabling strict mode
# Strict mode fails if an API mapping is missing, rather than passing it through.
ml_switcheroo convert ./src/ --out ./dst/ --strict

    2. Codebase Audit (audit)Â¶

    Analyze a codebase to check â€œTranslation Readinessâ€. This scans API calls and checks coverage against the Knowledge Base.

    ml_switcheroo audit ./my_project/ --roots torch

    3. Verification (ci)Â¶

    The CI command runs the built-in Fuzzer. It generates random inputs (Tensors, Scalars) based on Type Hints in the Spec, feeds them into both Source and Target frameworks, and mathematically verifies equivalence.

    # Run full verification suite on the Knowledge Base
ml_switcheroo ci

# Generate a lockfile of verified operations
ml_switcheroo ci --json-report verified_ops.json

    4. Knowledge Discovery (scaffold & wizard)Â¶

    Populate the Knowledge Base automatically by scanning installed libraries.

    # 1. Scaffold: Scan installed libs and generate JSON mappings via heuristics
ml_switcheroo scaffold --frameworks torch jax

# 2. Wizard: Interactive tool to manualy categorize obscure APIs
ml_switcheroo wizard torch

    ğŸ§  Advanced CapabilitiesÂ¶

    Functional UnwrappingÂ¶

    Frameworks like JAX require pure functions. ml-switcheroo automatically detects stateful imperative patterns (like drop_last=True in loops or in-place lists) and warns via the Purity Scanner. When converting Flax NNX (functional) to Torch (OO), it unwraps layer.apply(params, x) calls into standard layer(x) calls using Assign restructuring.

    State Injection (RNG Threading)Â¶

    When converting PyTorch (global RNG state) to JAX (explicit RNG keys), the engine:

    1. Detects stochastic operations (Dropout, Random init) via the Analyzer.

    2. Injects an rng argument into function signatures.

    3. Injects rng, key = jax.random.split(rng) preambles.

    4. Threads the key argument into relevant function calls.

    Intelligent Import ManagementÂ¶

    The Import Fixer does not just swap strings; it analyzes usage logic:

    • Removes unused source imports (import torch).

    • Injects required target imports (import jax.numpy as jnp) only if referenced.

    • Handles alias conflicts (import torch as t).

    ğŸ”Œ ExtensibilityÂ¶

    ml-switcheroo is designed to be extended without modifying the core engine.

    1. Add a Framework: Create a class inheriting FrameworkAdapter in src/ml_switcheroo/frameworks/.

    2. Add Definitions: Run ml_switcheroo scaffold or edit snapshots/{fw}_mappings.json.

    3. Add Logic: Write a localized hook in src/ml_switcheroo/plugins/ (e.g., for custom layer rewrites like MultiHeadAttention packing).

    See EXTENDING.md for a detailed guide.
