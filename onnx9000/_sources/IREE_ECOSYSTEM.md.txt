# Demos & Examples for `@onnx9000/compiler` (IREE/WVM)

> **Ecosystem Context:** `onnx9000` is a **Zero-dependency, WASM-First, and WebGPU-Native Polyglot Monorepo**. Through `@onnx9000/compiler`, it supports real-time transpilation and offline conversions across C++, MLIR, and `.wvm` targets without requiring heavy official bindings or native dependencies.

1. **[Tiny LLM in 20KB JS](https://github.com/onnx9000/tiny-llm-js)**
   A pure Vanilla JS + WebGPU standalone model executing an LLaMA model natively using `@onnx9000/core` and `@onnx9000/backend-web`.

2. **[Webcam Object Detection](https://github.com/onnx9000/webcam-yolo-standalone)**
   Real-time YOLOv8 rendering bounding boxes directly into HTML Canvas without any external dependencies, powered by `@onnx9000/backend-web`.

3. **Deno/Bun WVM Execution**
   Example showing how to run `.wvm` bytecode (generated by `@onnx9000/compiler`) in Deno via `deno run --unstable deno-wvm-example.js`.

4. **[Compiler Explorer](https://compiler.onnx9000.dev)**
   Interactive Web UI for pasting `.onnx` and seeing `.mlir`, `WGSL`, and `.wvm` updates in real-time.

5. **[Chrome Extension](https://github.com/onnx9000/chrome-wvm-extension)**
   Service worker embedding executing `.wvm` models offline directly in the browser background using the `@onnx9000/compiler` runtime.

6. **Cross Platform Parity**
   The exact `.wvm` binary evaluates perfectly bit-for-bit on Node.js, Deno, and Google Chrome.

7. **REST API Server**
   `@onnx9000/serve` example loading `.wvm` models and exposing an OpenAI compatible `/v1/chat/completions` endpoint.

8. **WVM Model Gallery**
   Pre-compiled `ResNet50.wvm`, `MobileNetV2.wvm`, and `TinyLlama.wvm` are available for zero-dependency deployment.

9. **Transition Docs**
   See `docs/TUTORIAL_MLIR_LOWERING.md` for understanding the AOT vs JIT execution models in the `@onnx9000` ecosystem.
