onnxmltools Replication & Parity Tracker¶

Description¶

This document tracks the complete reimplementation of onnxmltools within the onnx9000 ecosystem. The original onnxmltools relies heavily on native bindings for LightGBM, XGBoost, CatBoost, CoreML, and PySpark to perform conversions. Our onnx9000 reimplementation uses pure-Python parsers (reading raw JSON dumps, protobufs, or binary structures) to translate these diverse model types into ai.onnx.ml and ai.onnx topologies. This zero-dependency approach means you can convert a 10,000-tree XGBoost model or a complex SparkML pipeline entirely in a web browser using WASM or in a cold-start AWS Lambda function without installing multi-gigabyte ML frameworks.
